# Public Tester Call

AI Workbox is looking for a very small number of relevant testers.

This is not a broad launch.

## Who This Is For

You are a good tester if at least two are true:

- You use Windows as your main development machine.
- You use Node/npm or Next.js.
- You use Codex, Claude Code, Cursor, or another AI coding workflow.
- You have dealt with stuck `node.exe`, occupied ports, orphaned dev servers, or unclear local runtime state.
- You do not want Docker or a VM for every small dev command.

## What To Look At

Repo:

```text
https://github.com/skjshr/ai-workbox
```

Preview release:

```text
https://github.com/skjshr/ai-workbox/releases/tag/v0.1.0-preview
```

The Preview Pack is a self-contained win-x64 zip. It includes `bin/workbox.exe` and `try-smoke.ps1`.

## What I Need To Learn

Please answer any of these:

1. Is it clear what problem AI Workbox is solving?
2. Can you name a situation where you would run it?
3. Would you try `try-smoke.ps1`, `workbox doctor nextjs`, or `workbox run --name web -- npm run dev`?
4. Does the README make it clear that this is not a security sandbox?
5. Would a Japanese setup/troubleshooting pack be useful enough to pay for?

## Not Looking For

- Generic praise
- UI ideas before the CLI use case is proven
- Security sandbox expectations
- Enterprise endpoint/security claims
